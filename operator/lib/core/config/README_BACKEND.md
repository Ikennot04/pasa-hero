# Operator app – Firebase & Node.js backend

## Firebase

Firebase is initialized in `main.dart` using `firebase_options.dart` (generated by FlutterFire CLI). The operator app uses:

- **Firebase Core** – already initialized on startup
- **Firebase Auth** – available for operator login (wire your login form to `FirebaseAuth` or backend)

No extra setup needed if you already ran `flutterfire configure` and have `google-services.json` (Android) / `GoogleService-Info.plist` (iOS) in place.

## Node.js backend

The app talks to your Node.js backend via `BackendApiService` and `api_config.dart`.

### Base URL

- **Default:** `http://localhost:3000`
- **Override:**  
  `flutter run --dart-define=BACKEND_URL=https://your-api.com`  
  or for Android emulator:  
  `flutter run --dart-define=BACKEND_URL=http://10.0.2.2:3000`

### Usage

```dart
import 'package:admin/core/services/backend_api_service.dart';

final api = BackendApiService();

// Health check
final ok = await api.checkHealth();

// GET
final r = await api.get('/api/operators/me');
if (r.success) { ... }

// POST (e.g. login)
final r = await api.post('/api/auth/operator/login', body: {'email': email, 'password': password});
```

Your Node.js server should expose endpoints (e.g. `/api/health`, `/api/auth/operator/login`) and use the same Firebase project if you need to verify Firebase ID tokens.
